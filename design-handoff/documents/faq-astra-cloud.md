# FAQ Astra Cloud — фінальна версія

Джерело: аналіз тікетів техпідтримки HostBill (~270 реальних звернень, 2023-10 — 2026-07),
узгоджено та відредаговано командою.

Класифікація питання:
- **[Компанія]** — стосується компанії в цілому (оплата, правила, колокація, підтримка тощо)
- **[VPS]** — стосується послуги VPS/VDS
- **[Дедік]** — стосується виділених серверів (Dedicated Server)
- **[Компанія + VPS + Дедік]** — питання, що стосується кількох напрямків одночасно

## 1. Компанія в цілому

**Де розташовані ваші дата-центри?** [Компанія]
Основна локація — Львів, Львівська область (2 дата-центри). Для міжнародного транзиту/пірингу є присутність у Варшаві (Польща).

**Яка у вас цінова політика для колокації власного обладнання?** [Компанія]
Доступні формати від 1U до повної стійки (42U), включно з 1/3 та 1/2 стійки. Живлення — або фіксована потужність з щомісячною оплатою, або облік по лічильнику з коефіцієнтом PUE ≈1.9. Резервування живлення — ДБЖ + генератор, за потреби — два незалежні ДБЖ (A+B). Канали зв'язку — від 10 Мбіт/с до 10 Гбіт/с, можливе оголошення власного ASN та підмереж клієнта через BGP. Комерційну пропозицію готують індивідуально (зазвичай 1–3 робочих дні).

**Чи можна замовити IP-адреси окремо, без серверу (IP-лізинг)?** [Компанія]
Ні. Пул IP-адрес надається лише як додаткова опція для клієнтів колокації/дедікованих серверів, окремо в оренду не здається.

**Які способи оплати ви приймаєте?** [Компанія]
Базово — банківські картки через LiqPay. PayPal і Stripe напряму не підтримуються (обмеження для українського бізнесу). Для міжнародних клієнтів додано оплату криптовалютою (USDT TRC20) та сервіс Paddle/PaddlePay, через який також доступний PayPal.

**Чому не проходить оплата міжнародною/іноземною карткою?** [Компанія]
LiqPay вимагає проходження 3D Secure, і частина закордонних банків (Revolut, Wise, Payoneer, картки не з України) цю перевірку не підтримують — платіж автоматично відхиляється на стороні банку. Рекомендація підтримки: спробувати іншу картку (бажано українського банку) або скористатись криптовалютою/Paddle.

**Чи є реферальна або партнерська програма?** [Компанія]
Так! Ми запустили партнерську програму для тих, хто готовий рекомендувати Astra Cloud і отримувати за це винагороду. Усі умови та механіка приєднання — на сторінці **/partnerka**. Якщо у вас особливі умови співпраці на думці або проєкт з великою аудиторією — звертайтеся до служби підтримки, обговоримо індивідуально.

**Які правила використання серверів (TOS)?** [Компанія]
Заборонено: спам, сканування портів, DDoS-атаки (вхідні й вихідні), розміщення незаконного контенту, порушення авторських прав, зломи/фішинг/ботнети/шкідливе ПЗ, продаж зброї/наркотиків/підроблених документів. За скаргу або потрапляння IP у Spamhaus через дії клієнта сервер може бути заблокований без повернення коштів. Торренти самі собою дозволені, якщо це не порушує авторське право і не викликає скарг.

**Як ви реагуєте на скарги про зловживання (abuse/DMCA/фішинг)?** [Компанія]
Скарги (зокрема від Cloudflare Trust & Safety чи інших ДЦ) опрацьовуються оперативно: сайт/акаунт порушника блокується, скаржнику надсилається підтвердження. Партнерам-реселерам компанія просить пересилати повний текст скарги для прозорості.

**Чи надаєте тестовий/пробний період?** [Компанія]
Так, надаємо — умови розглядаємо індивідуально залежно від послуги та ваших потреб. Звертайтеся до служби підтримки, і ми підберемо оптимальний варіант.

**Які години роботи технічної підтримки?** [Компанія]
За телефоном підтримка доступна з 9:00 до 22:00. У месенджерах та по електронній пошті — цілодобово, 24/7.

**Чи берете участь у державних закупівлях (Prozorro)?** [Компанія]
Так, компанія отримує та опрацьовує тендерні запрошення (зокрема на послуги колокації для держпідприємств).

## 2. VPS

**Де фізично розташовані VPS-сервери і яка мережа?** [VPS]
Львів. Основний пул адрес — 91.245.76.0/24; для окремих випадків (наприклад, заміни заблокованих IP) використовується додатковий пул 95.141.245.0/…

**Чим відрізняються тарифи VPS (Nano/Micro/Standard/Premium/Maximum)?** [VPS]
Різниця в CPU, RAM, обсязі диска, швидкості каналу (100 Мбіт/с на молодших → 1–2.5 Гбіт/с на старших) та лімітом трафіку. Починаючи з VPS Standard трафік безлімітний; на Nano/Micro є ліміт.

**Що відбувається після вичерпання ліміту трафіку?** [VPS]
Швидкість падає до 10 Мбіт/с до кінця розрахункового періоду. Можна докупити додатковий обсяг (орієнтовно 100–150 грн за 1 ТБ, діє лише до кінця поточного періоду).

**Чому я не можу зайти по SSH під root?** [VPS]
Це свідоме рішення з міркувань безпеки, а не недогляд і не те, що ми не хочемо змінювати. Сучасні cloud-init образи (Ubuntu 22+, Debian 12+, AlmaLinux) вимикають прямий вхід під root за замовчуванням — це стандартна практика захисту сервера від брутфорс-атак і несанкціонованого доступу. За потреби ви можете самостійно активувати root-вхід: `sudo passwd root`, потім у `/etc/ssh/sshd_config` виставити `PermitRootLogin yes` і перезапустити `sshd`.

**Як перевстановити ОС або обрати іншу?** [VPS]
Через особистий кабінет (функція "Reset"/перевстановлення).

**Чи можна замовити кастомну конфігурацію VPS (нетипове співвідношення CPU/RAM/диска)?** [VPS]
Повноцінного конфігуратора немає. Точково можна докупити додатковий диск або RAM за окремим прайсом, або узгодити нестандартний тариф індивідуально через підтримку (так з'явився, наприклад, окремий тариф "16 core / 32 RAM / 512 GB" для одного з клієнтів).

**Чи можна замовити додаткові IP на VPS?** [VPS]
Так, можна замовити додаткові IP — кількість залежить від тарифу. На VPS Standard доступно до +2 додаткових IP, на VPS Premium — до +4.

**Сервер недоступний / офлайн — як швидко це вирішується?** [VPS]
Типове рішення — перезавантаження з боку підтримки. Якщо мережевий доступ відсутній, радять користуватись "Терміналом"/VNC в особистому кабінеті — це працює навіть без мережі.

**Як скасувати/видалити VPS?** [VPS]
У кабінеті → розділ "Білінг" → подати запит на видалення. Якщо є неоплачені інвойси — спершу потрібно їх закрити чи звернутись у підтримку. Після запиту VPS ще кілька днів доступна для відновлення, потім видаляється остаточно.

**Чи повертають гроші за передплачений, але невикористаний період?** [VPS]
Так, повернення можливе, але лише після місяця користування послугою. Залишок за невикористаний період рахуємо за запитом і зараховуємо на баланс.

**Чи є автоматичне списання оплати з картки?** [VPS]
Функція автосписання з прив'язаної картки на момент дослідження — на етапі тестування. Стандартно щомісяця формується інвойс, який або оплачується вручну, або автоматично закривається з попередньо поповненого балансу.

**IP потрапив у Spamhaus (PBL) — це проблема?** [VPS]
Здебільшого йдеться про Spamhaus PBL (Policy Block List) — стандартна практика для нових/динамічних діапазонів хостингів, вона не означає, що IP "заблокований за спам", і не впливає на роботу сервісів, окрім вихідної пошти (SMTP). Підтримка подає запит на delisting вручну і зазвичай знімає лістинг протягом 10–20 хвилин.

**Чи можна передати VPS іншому клієнту (зміна власника)?** [VPS]
Так, підтримка може перенести послугу на обліковий запис іншого клієнта.

**Як налаштувати RDNS/PTR запис для VPS?** [VPS]
Самообслуговування в кабінеті відсутнє — запис вносить мережевий адміністратор за зверненням у підтримку.

## 3. Виділені сервери (Dedicated Server)

**На якому обладнанні базуються дедіковані сервери?** [Дедік]
Виключно серверне обладнання рівня enterprise — Dell R640/R340 з процесорами Intel Xeon Gold/Silver, серверна RAM (ECC), диски SAS SSD/HDD. Політику змінили: десктопні процесори (i3/i5/i7/i9) більше не використовують для збірки нових серверів, хоча старі згадки про них ще залишаються на сайті.

**Скільки часу займає підготовка (збірка) сервера?** [Дедік]
Заявлено до 2 тижнів; на практиці, якщо потрібні комплектуючі вже є в наявності — 2–7 днів.

**Які додаткові IP доступні для виділеного серверу?** [Дедік]
Тільки пулами: /30 (1 корисна IP серед broadcast/gateway), /29 (5 корисних), /28 тощо — з відповідною вартістю за розмір пулу.

**Чи надається ліцензія Windows на виділеному сервері?** [Дедік]
Ні, ліцензії Microsoft (SPLA чи інші) компанія не продає й не активує. Можна встановити тільки trial-версію або власну ліцензію клієнта.

**Як я можу самостійно керувати "залізом" (перевстановити ОС тощо)?** [Дедік]
Через віддалену консоль управління (iDRAC у Dell R640, або аналог ILO), доступ до якої надається після збірки сервера.

**Root доступ на дедіку теж вимкнено?** [Дедік]
Так, на образах AlmaLinux root вимкнений за замовчуванням так само, як і на VPS; активується вручну через sudo.

## 4. Питання на стику кількох напрямків

**Оплата іноземними картками / PayPal / Stripe / криптовалюта** [Компанія + VPS + Дедік]
Однакова відповідь для всіх послуг: LiqPay не завжди приймає закордонні картки через вимогу 3D Secure; PayPal/Stripe напряму недоступні через обмеження для українського бізнесу; альтернатива — крипта (USDT TRC20) або нещодавно підключений Paddle/PaddlePay (в т.ч. PayPal через нього).

**Root SSH-доступ вимкнено за замовчуванням** [VPS + Дедік]
Стандартна поведінка cloud-init образів незалежно від типу послуги — свідоме рішення з міркувань безпеки. Потрібно активувати вручну (`passwd root` + `PermitRootLogin yes` + рестарт sshd).

**IP у списку Spamhaus PBL** [VPS + Дедік]
Та сама політика і те саме пояснення (PBL ≠ бан за спам) застосовується як до VPS, так і до виділених серверів; підтримка знімає лістинг за запитом.

**Налаштування RDNS/PTR запису** [VPS + Дедік]
В обох випадках — тільки через звернення в підтримку, самостійно в кабінеті не редагується.

**Скасування послуги та повернення коштів** [Компанія + VPS + Дедік]
Загальний принцип для всіх послуг — запит подається через "Білінг" в кабінеті, точний момент запиту є точкою відліку для розрахунку часткового повернення коштів.
